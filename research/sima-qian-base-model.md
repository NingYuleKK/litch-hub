# 太史公 base model 研究笔记

> 从《史记》到大语言模型的叙事先验

**来源**：2026-03-03 Litch 与 Claude（小克）、GPT（Root）的对话  
**触发**：刘邦临终托孤 vs 诸葛亮临终托孤的同构结构

---

## 核心发现

### 触发场景
刘邦临终——吕后三问，点名曹参→王陵（陈平辅）→周勃，最后截断："此后非而所知也。"  
诸葛亮临终——李福三问，点名蒋琬→费祎，最后沉默。  

**问题**：隔四百年，结构几乎一模一样。巧合？套路？

### 结论
不是纯巧合，也不是纯套路——是三层叠加：
1. **史书叙事语法**：史家有"临终托付"模块，天然需要"层层递进+最后留白"的节奏
2. **政治现实产生相似结构**：人治体制下接班人安排本就是有限深度的递归问题
3. **截断的含义不同**：刘邦是权力边界（开创者的清醒），诸葛亮是命运尽头（人才断层的悲剧性沉默）

---

## 核心命题：司马迁 = Base Model

### 结构化 token 分层
- 本纪 / 世家 / 列传 = 类别映射系统（什么级别的人用什么结构）
- 酷吏列传、游侠列传 = domain-specific embedding

### 叙事模板（prompt templates）
| 场景 | 模板 |
|------|------|
| 临终托孤 | 三问两答，最后截断 |
| 大战 | 铺垫→军阵对比→天时地利→人物心理钩子 |
| 酷吏 | 简短、锋利、冷血 |
| 侠客 | 突然出手、重诺、轻生死 |

**结论**：司马迁不是在写历史，而是在训练一种"历史生成器"。后世二十四史 = 在他的权重上 domain-specific fine-tune。

---

## 核心困境：世界的 pattern vs 模型的 prior

**问题**：我们永远无法彻底分清——  
是现实本来有这个 pattern，  
还是叙述者的模型把现实压缩成了这个 pattern？

**具体例子**：
- 刘邦可能只是零散说过"萧何可、曹参可、王陵不错"，史书用一场床前问答把碎片压缩
- 诸葛亮对蒋琬费祎的信任可能也不是一次说完，陈寿拼成了连续对话

**AI 同构**：模型训练时，现实高频结构 + 训练偏置，两者纠缠在一起无法剥开。

---

## 史学反向工程三步（GPT 提炼）

1. **识别模板**：看到结构，判断是"太史公系"还是"班固系"
2. **拆分原始输入 vs 自动补全**：哪些是史实核，哪些是叙述者的补全？
3. **追问生成路径**：这段文字是怎么被生成的？

**三连问校准工具**：
1. 这个场景有没有强烈"文学感"？（节奏、对白是不是太工整）
2. 有没有前代经典里几乎一模一样的结构？
3. 后世发展是否"完美印证"了这段预言式对白？

→ 1+2+3 同时满足 = 进入"读模型"模式，而非"读录音"模式
