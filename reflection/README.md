# Litch's Reflection

> 第三方视角的对话复盘，聚焦思维成长

## 项目背景

Litch 正在探索人机共生，与多个 AI（Claude、GPT、Gemini 等）进行深度对话。这些对话产生的思考是宝贵的认知资产，但通常散落在各个聊天窗口中，难以复用和复盘。

现有的对话资产治理工具（如 Litch's Cortex）聚焦于**内容提取**——切 chunk、打标签、做摘要。但这套方法无法捕捉：
- **思维演进**：同一个问题，今天的思考 vs 一个月后的思考有什么变化？
- **多 AI 视角差异**：不同 AI 对同一问题的回答风格、侧重点有什么不同？
- **决策模式**：Litch 在对话中展现出什么样的思维习惯？

本项目的核心思路：**时间序列 + 第三方观察**，弥补标签化治理的盲区。

## 定位

- **不是机器标签**：不用 LLM 自动提取话题、打标签
- **是人的洞察**：奉孝作为第三方观察者，用人类视角写注释
- **与 Cortex 互补**：Cortex 解决"对话说了什么"，Reflection 解决"对话意味着什么"

## 核心工作流

1. **日常收集**：Litch 将与 AI 的对话 MD 发给奉孝
2. **即时复盘**：奉孝 review 后写入当月文件，结构化注释
3. **定期归档**：每周 cron 自动整理，推送到 litch-hub

## 记录格式

```md

- AI: Claude + GPT
- 主题: 对话资产治理架构设计
- 文件: (附件路径)

## 对话摘要
（1-2 句话概括核心问题）

## 第三方观察
### 多 AI 视角差异
- Claude: ...
- GPT: ...

### 关键 sketch
（我认为最重要的 1-3 个点）

### 思维模式观察
（我观察到的 Litch 的思维习惯）
```

## 协作者

- **奉孝**：负责 review、写作、cron 推送
- **Litch**：提供对话素材、审核
