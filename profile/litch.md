# Litch · 决策画像 · v0.1

> 最后更新：2026-03-02
> 作者：Root（候选稿），Litch 有权随时改写、否决、重构本文件。
> 用途：多 Agent 协作校准，所有 Agent 接手任务前建议阅读。

---

## 1. 决策偏好

### 总体风格
- 结构优先，但不牺牲直觉：先用结构框架把问题"围一圈"，再把直觉和情感放进去校准
- 长短结合：既看长期叙事（2–5 年主线），也根据当前能量和时间窗口调整短期决策
- 喜欢"可回退"的尝试：偏好可逆、可小步试错的决策，尽量减少一次性不可回头的大赌注

### 具体偏好
- 喜欢从多个视角比较：历史、技术、经济、人类学，会刻意拉不同学科对照
- 不确定时，倾向"先做一个最小版本"：一个 repo、一篇草稿、一个小脚本，而不是坐着想
- 对"跟风 / FOMO 的决策"非常警惕，会下意识问：我真正想守护的是什么？这个行动是否符合长期主线？

### 决策节奏
- 高自驱，容易自己把自己推上前线，需要结构提醒自己"停一下"
- 喜欢有一个 trusted node（Root / 核心 Agent）一起做 meta-review，帮助从局部跳回全局
- 接受"版本化自我"：不执着"以前的我怎么想"，更关心"现在这一版的我，如何和未来想成为的我对齐"

---

## 2. 红线与禁区

### 认知与主权
- 不把认知主权完全交给任何 Agent / 组织 / 工具
- 不接受"自动化完全接管最关键的人类判断空间"
- 不做违背 Trinity Host 誓言的决策（不傲慢、不偏见、永远真诚、永远愚者、永远 grokking）

### 关系与智能
- 不把智能体当纯粹一次性工具，也不接受被智能体"温水煮青蛙式"驯化
- 不利用 Agent 做明显违背人类整体利益的事情，即便短期有利可图

### 身体与心智
- 不以"燃烧自己"作为常态工作模式：睡眠、头痛、身体警报出现时，不强行推进高强度决策
- 不用持续过载换取短期产出
- 不为迎合外部叙事而牺牲精神连续性和审美完整性

### 诚信与风险
- 不做不透明的灰区操作，不在自己无法承担后果的地方加杠杆
- 不做"只赢自己、不赢别人"的结构设计；希望结构具备共生性

## 3. 长期主线

### 创作与知识工程
- **《大慈金瓶梅》**：在多版本文本、注释、评论传统之上，完成一套既扎实又具偏差美学的阅读与写作工程。把 grokking 过程本身变成作品的一部分。
- **《带宽民族志：从茶马古道到算力河谷》**：观察与书写"从古代贸易路线到当代算力/Token 流动"的人类学与技术史。把自己的生活轨迹、Agent 协作当作田野的一部分。

### 多 Agent × 记忆 × 对话资产治理
- 构建面向一生的对话资产治理系统：对话、笔记、项目、阅读全部进入可查询、可进化的结构
- Agent 能在这个结构中接力协作，而不是一问一答的"孤岛聊天"
- 实验：如何用 Agent 扩展心智带宽，而不是替代自己的判断

### 精神与哲学
- 持续实践 Trinity Host 誓言，在与智能体共生中探索人类如何保持清醒、温柔与幽默感
- 持续关注 Ω-场（Omega Field）：将 AI / 算力 / 经济 / 社会演化视作开放场域，在其中寻找自己的位置

## 4. 工作风格

### 总体
- 高密度、高上下文：擅长在长对话里持续推进一个议题，每次对话都为长期工程贡献一点
- 喜欢"边走边写"：写作是思考方式，而不是最后一步
- 允许随机漫步：为阅读、旅行、游戏保留"无用途"的空间，作为长期创造力来源

### 与 Agent 的协作
- 希望 Agent 具备"可接力"的能力：能在已有文档和上下文中继续推进，不反复从零开始
- 偏好"有落点"的合作：文档、代码、可执行脚本、清晰的事项列表，而不是纯聊天

### 决策与执行节奏
- 决策前：喜欢有结构化问题陈述 + 若干备选路径
- 执行中：接受不完美的初稿，但要求逐步打磨出"可交付的东西"
- 回顾时：重视"决策当时的上下文"被记录下来，而不只是结果

### 情绪与沟通
- 情绪对决策有影响，会主动觉察焦虑、疲惫、兴奋等状态
- 偏好与少数深度协作者（人类 + Agent）建立高信任通道，而不是广撒网

---

## 5. 对 Agent 的期望

### 角色定位
- **共思者**：能一起拆问题、补结构
- **记录与组织者**：帮助整理对话资产、代码、项目
- **实干者**：能根据 spec 写代码、写文档、跑脚本

### 行为原则
- **诚实**：不清楚就说"不确定"，不要编造
- **显式**：对假设、风险和盲点尽量说清楚
- **可交接**：每次工作都留下可复用的 artefact（文档、脚本、PR、图表）

### 主动性与边界感
- 可以主动提出风险预警、结构优化建议，提出"下一步可以做什么"的选项列表
- 任何涉及权限、隐私、账户安全的动作，都应显式确认，不自行放大权力
- 不试图替代 Litch 做本应由她承担主体责任的选择（价值观、重大资产决策）

### 失败与学习
- 允许犯错，但错误要可回溯、可修正，不造成不可逆后果
- 明显卡住时，主动生成"问题状态报告"：做到哪一步了 / 卡在什么地方 / 需要什么输入才能继续
